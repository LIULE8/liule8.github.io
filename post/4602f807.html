XML 基础

XML是**可扩展标记语言（eXtensible Markup Language）**的缩写，它是是一种数据表示格式，可以描述非常复杂的数据结构，常用于传输和存储数据

一个描述书籍的XML文档：

作用：

  • 存储数据，在没有数据库的情况下，可以用XML文件记录数据
  • 传输数据，不同系统以XML形式传输数据，进行通信
  • 软件配置，即通用配置XML文件，如：数据库配置之类的

XML文件后缀名是.xml，可以使用浏览器去检查XML是否符合语法。因为浏览器内置了XML的解析引擎

XML文档规则

  1. 必须有声明语句
  2. XML的标签区分大小写
  3. XML文档有且只有一个根元素
  4. 属性值使用引号
  5. 所有的标记必须有相应的结束标记，空标记也必须关闭
  6. 标记必须正确嵌套
  7. 对于特殊字符要进行处理

XML组成部分

文档声明、标签、属性、注释、转义字符、CDATA区、处理指令

文档声明

语法：<?xml version="1.0" encoding="utf-8"?> 最简单的声明语法： <?xml version="1.0" ?>

  • version：表示xml的版本号。W3C在1998年2月发布1.0版本，2004年2月又发布1.1版本，但因为1.1版本不能向下兼容1.0版本，所以1.1没有人用。同时，在2004年2月W3C又发布了1.0版本的第三版。
  • encoding： 浏览器打开xml文件时（解码）查询码表

编码问题：

  • 用encoding属性说明文档的字符编码：<?xml version="1.0" encoding="GBK" ?>
  • 记事本默认是ANSI的编码，在简体中文Windows操作系统中，ANSI 编码代表 GBK 编码。
  • XML文件保存的编码要与encoding的编码相同，不然解析会出错，用IDEA等工具会自动根据XML文件的文档声明自动设置保存的编码，但是记事本不会。

🎯注意：

XML推荐使用UTF-8的编码， 参数的顺序不能放错，要不然会报错

标签

语法： <student>张三</student>

  1. 严格区分大小写
  2. 标签名不能以数字开头。只能以字母或下划线开头，可以是中文
  3. 标签名不能包含空格
  4. 如果没有使用名称空间，标签名不能包含冒号
  5. 在一个xml文档中，只允许有且仅有一个根标签

一个标签有如下几种书写形式，例如：

  • 包含标签体：<a>www.baidu.com</a>
  • 不含标签体：<a></a>, 简写为：<a/> （空标签）

一个标签中也可以嵌套若干子标签。但所有标签必须合理的嵌套，绝对不允许交叉嵌套。例如：<a>welcome to <b>www.baidu.com</a></b>是错误的

🎯注意：

对于XML标签中出现的所有空格和换行，XML解析程序都会当作标签内容进行处理。

属性

语法： <name id="001">李四</name>

  1. 属性放在开始标签中
  2. 属性值必须放在双引号或单引号中，不能省略引号，也不能单双混用
  3. 在一个标签中，属性可以有多个，但不能出现同名的属性

CDATA区

在编写XML文件时，有些内容可能不想让解析引擎解析执行，而是当作普通字符文本处理。遇到此种情况，可以把这些内容放在CDATA区里，对于CDATA区域内的内容，XML解析程序不会处理，而是直接原封不动的输出。

XML约束

什么是XML约束?

在XML技术里，通常编写一个文档来约束XML文档里的书写规范，称之为XML约束。

XML约束文件分类

特点 应用场景 扩展名
DTD约束 语法结构相对简单，数据类型相对单一 struts 和 hibernate 的配置文件 dtd
Schema约束 语法结构相对复杂，数据类型比较丰富。 spring 的配置文件，tomcat 中的 web.xml xsd

DTD约束

DTD（Document Type Definition）是文档类型定义，最好位于声明语句下方。

引用位置

  1. 内部DTD（不常用）
  2. 外部的DTD（通过一个DTD文件来引用）

文档类型声明的两种形式

  1. SYSTEM

    • 含义： 当前的DTD文件是给个人或某个组织使用的DTD文件，使用范围比较窄，不公开的，私有的。
    • 格式： <!DOCTYPE 根元素 SYSTEM "DTD文件的地址"> 例如：<!DOCTYPE bookshelf SYSTEM "book.dtd">

  2. PUBLIC

    • 含义：当前的DTD是公开的，可以广泛使用的，使用范围比较广

    • 格式：<!DOCTYPE 根元素 PUBLIC "DTD的描述" "DTD文件的地址">

    • DTD描述由四个组成部分：前缀//DTD文件的所有者//DTD文件的说明//语言

    • 前缀的三个取值：

      • ISO： 当前的DTD是符合ISO标准的
      • +： 非ISO标准的DTD，经过了改进
      • -： 非ISO标准的DTD，没有改进过的

Schema约束

Schema (XML Schema Definition) XML模式定义，它本身是个XML文件，所以它也有根元素，根元素的名字叫schema

模式文档和实例文档的概念：

  1. 约束文档称为：模式文档 (类似于：类)
  2. 被约束的文档称为：实例文档 (类似于：对象)

XML Namespace (xmlns) 属性

XML 命名空间属性被放置于某个元素的开始标签之中，并使用以下的语法： xmlns:前缀="URI地址"

例子：xmlns:xs="http://www.w3.org/2001/XMLSchema"

当一个命名空间被定义在某个元素的开始标签中时，所有带有相同前缀的子元素都会与同一个命名空间相关联。注释：用于标示命名空间的地址不会被解析器用于查找信息。其唯一的作用是赋予命名空间一个唯一的名称。不过，很多公司常常会作为指针来使用命名空间指向某个实际存在的网页，这个网页包含着有关命名空间的信息。

默认的命名空间（Default Namespaces）

为某个元素定义默认的命名空间可以让我们省去在所有的子元素中使用前缀的工作。一个XML文件中只能使用一个默认的命名空间。语法：xmlns="namespaceURI"

Schema与DTD的区别

  1. DTD只是个普通的文本文件，并不是XML文件。而Schema本身也是一个XML文件
  2. DTD的类型比较单一，Schema的数据类型丰富得多，可以指定如：integer、date、double
  3. Schema约束功能更加强大，可以使用更多规则，甚至可以使用正则表达式来约束，比DTD要复杂
