Java 集合类详解

ArrayList

ArrayList 可以自动扩容和缩容的集合类，基于数组实现，允许元素为null；允许重复的数据；非线程安全；

Vector 是线程安全的，但是有较多缺点，不推荐使用

继承结构

分析：

  1. 为什么 AbstractList<E> 已经实现了 List<E> ，ArrayList<E> 还需要再实现 List<E> ？

    stackOverFlow 有个提问 : 其中一个回答是 作者Josh Bloch 说这是一个mistake，写的时候觉得会有用处，但其实没什么作用，但因为没有影响，所以留到现在

  2. 为什么需要有一个 AbstractList<E> 抽象类 ?

    抽象类可以实现一些通用的方法，然后实现类再加特有的方法；代码更加简洁，减少重复代码

  3. RandomAccess 接口：标记性接口，用来快速随机存取；如果实现了该接口的话，那么使用普通的for循环来遍历，性能更高；如果没有实现该接口的话，使用Iterator来迭代，这样性能更高

  4. Cloneable接口：可以使用 Object.Clone()

  5. Serializable接口：序列化接口，表明该类可以被序列化（将类变成字节流，或者将字节流变成类）

add 方法

boolean add(E e)

默认直接在末尾添加元素

ensureCapacityInternal 确定内部容量的方法

calculateCapacity 计算容量

ensureExplicitCapacity 计算的当前容量跟实际容量比，如果不够则扩容

grow 拓展数组大小

hugeCapacity 获取ArrayList可以容纳的最大值

void add(int，E)

在特定位置添加元素，也就是插入元素

rangeCheckForAdd 判断index是否合理

add 总结

正常情况下会扩容1.5倍，特殊情况下（新扩展数组大小已经达到了数组规定的最大值 Integer.MAX_VALUE - 8）则只取整型的最大值（Integer.MAX_VALUE）。

remove 方法

E remove(int)

通过删除指定位置上的元素

boolean remove(Object)

fastRemove 和 remove(index) 的实现几乎一样， remove(index) 多了检查index是否合法

void clear()

将 elementData 中每个元素都赋值为null，等待垃圾回收将这个给回收掉，所以叫clear

boolean removeAll(Collection)
boolean batchRemove(Collection, boolean)

用于两个方法，一个是removeAll：清除集合元素；另一个是retainAll：测试两个集合是否有交集

remove 总结

remove 函数用户移除指定下标的元素，此时会把指定下标到数组末尾的元素向前移动一个单位，并且会把数组最后一个元素设置为null，这样是为了方便之后将整个数组不被使用时，会被GC，可以作为小的技巧使用。
